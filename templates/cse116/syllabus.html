Course Description

You know the basics of programming. You're proficient in variables, functions, control flow, data structures, file IO, networking, databases, and security. With these concepts you can train a computer to complete nearly any task you can imagine. So what's next?

Writing large programs

You will learn how to write and organize large pieces of software. To this end, in CSE116 you will study:

  • Program Execution
  • Object-Oriented Programming
  • Functional Programming & Concurrency
  • Data Structures & Algorithms

While studying these topics we will have a strong focus on how your programs work, not just that they work.

Learning Objectives

In CSE116 you will learn concepts related to the following 4 learning objectives. Being successful in CSE116 requires completion of all 4 of these objectives. Each objective is divided into 3 parts:

  • Read - Given a program related to the learning objective, trace through its execution and describe what the computer does at each step. This will be assessed through the quizzes during the learning objective verification labs.
  • Explain - Communicate verbally about the concepts of the learning objective. You must demonstrate an understanding of these concepts as well as comprehensive knowledge of the related terminology. This will be assessed through interviews during the learning objective verification labs where you will be asked questions about these concepts.
  • Write - Given a real-world problem statement, write a program that solves the problem. This will be assessed through the coding projects throughout the course. Each project will explain the criteria for completing the write portion of the learning objective.

Learning Objective 1 - Program Execution

In CSE115 there was a focus on writing programs to accomplish specific tasks. In CSE116 we will focus on how those programs run. We will look at how memory is managed and how to automate the testing of programs.

A student has completed this objective if they are able to:

  • Read: Trace through a program and track the state of stack memory throughout its execution
  • Explain: Describe the concepts of stack memory, stack frame, variable scoping, and unit testing, while reflecting on their importance in the programs we write
  • Write: Write methods using Scala and write automated tests to verify the functionality of such methods

Learning Objective 2 - Object-Oriented Programming

Instead of writing programs as a collection of variables and functions, with object-oriented programming we will group these variables and functions into classes and objects. This allows developers to reason about their programs at a higher level of abstraction.

A student has completed this objective if they are able to:

  • Read: Trace through the execution of programs that use inheritance and polymorphism while tracking the state of stack and heap memory
  • Explain: Describe the concepts of classes, inheritance, and polymorphism and explain how they can be used to help us write software
  • Write: Implement programs that change behavior based on the type of an object stored in a variable by applying the concepts of inheritance and polymorphism

Learning Objective 3 - Functional Programming & Concurrency

Functional programming is based on functions with clearly defined input/output behavior. If a function is called multiple times with the same input, it must always have the same output regardless of the state of the program. This consistent behavior makes functional programs suitable for multi-threading and large apps that need to be distributed across multiple machines (ex. YouTube is much too large run on a single machine).

A student has completed this objective if they are able to:

  • Read: Trace through the execution of programs that use recursive methods/functions, first-order functions, immutable types, and concurrency while tracking the state of stack and heap memory
  • Explain: Describe the concepts of recursion, first-order functions, immutability, and concurrency while explaining their implications for software that applies these concepts
  • Write: Implement recursive methods/functions, methods that take a function as a parameter, methods that return a function, and programs that use immutable types

Learning Objective 4 - Data Structures & Algorithms

We will study several new approaches for storing data and computing with that data. Primarily we will see linked-lists, trees, and graphs while exploring applications of each structure.

A student has completed this objective if they are able to:

  • Read: Trace through the execution of programs that use linked-lists, trees, and graphs
  • Explain: Describe the structure of linked-lists, trees, and graphs and the implications of these structures including applications of each
  • Write: Write programs that use linked-lists, trees, and graphs as well as implement new algorithms to expand the functionality of each data structure

Learning Objective Completion

You must demonstrate that you've completed all 4 learning objectives in order to pass this class.

All decisions relating to learning objective completion will be made by the course staff using your performance in the course as evidence. Keep in mind that your goal in this class is to convince a reasonable human that you have completed the read, explain, and write parts of all 4 learning objectives. You will not fail this class due to a grading script not awarding you completion for an objective that you have rightfully completed. The opportunities below will streamline most of the grading decisions, but if you are not able to complete the learning objectives through these opportunities alone, we will discuss other ways for you to demonstrate completion. Your job is to focus on your education. We will worry about assessing whether or not you've learned what you need to learn from this course. Do not worry if you miss an opportunity. As long it's clear that you are putting your effort of at least 12 hours/week, you will be given more opportunities.

You will have at least the following opportunities to demonstrate completion of the 3 parts of each objective. One you earn completion for all 3 parts, you have completed the corresponding learning objective.

  • Read - Lab Quiz
    • Each learning objective will include a code reading quiz that you will take during lab. To complete the read portion of a learning objective, your quiz submission must demonstrate that you can read code as specified in the learning objective portion of the syllabus.
    • There are no grades for these quizzes. All completion decisions will be made by a human after reviewing your submission.
    • There are 2 lab quizzes on the schedule for each learning objective. If the course staff verifies that you've completed the read portion of a learning objective after you take the first quiz, you do not have to take the quiz during the second chance lab.
  • Explain - Lab Interview
    • During each learning objective lab, while you are taking your code reading quiz, you will have a 10 minute interview with a TA. The TA may ask any questions they deem appropriate in order to assess that you can explain the concepts as specified in the learning objective portion of the syllabus. The TA will submit a summary of your answers that will be reviewed by the course staff.
    • As with the lab quizzes, there are no grades for these interview. After human review of your answers, it will be decided whether or not you completed the explain portion of the learning objective through your interview performance. If you complete the interview during the first lab for a learning objective you do not have to interview again during the second chance lab. (If you completed both the quiz and interview during the first lab, you do not need to attend the second chance lab)
    • Your TA does not make decisions for learning objective completion. Do not pressure your TA into marking you for completion. Likewise, your TA cannot tell during the interview whether or not you passed since they will not know. The course staff will weigh all evidence while reviewing the reports from all TAs and efforts will be made to neutralize the variance between TAs (ie. Your completion of learning objectives should not be affected by which TA you were assigned for an interview).
  • Write - Coding Projects
    • Each learning objective will contain a variety of objectives that are contained in the programming projects throughout the semester. These projects will contain a combination of lecture objectives and application objectives, both of which involve writing code. To complete the write portion of a learning objective, you must complete the lecture objectives of the corresponding project.
    • Lecture objectives will be introduced in the lecture that covers the material needed to complete that objective. It is recommended that you start each of these objectives immediately after, or during, the lecture that introduces them and complete them before the next lecture starts.
    • If you earn credit for all required objectives in AutoLab you will have completed the write portion of that learning objective. In rare cases, some objectives may be waived after human review as long as you have demonstrated that you can write code as specified in the learning objective portion of the syllabus (ie. Do not stress about obscure errors in your AutoLab feedback as long as you know your code works and you have comprehensive unit testing. If your testing is anything less than amazing, do not expect a waiver).

Learning Objective Checkpoints and Recovery Plans: You are expected to complete all three portions of a learning objective before the checkpoint for that objective. All checkpoints are listed on the course schedule. If you do not complete an objective before its checkpoint, and it's clear that you've been putting in your >=12 hours/week, don't panic. All of your work will be reviewed and we will discuss a recovery plan for you to have an opportunity to makeup the parts of the objective that you missed. This may include more time to complete the coding objectives, another interview opportunity, and/or another quiz to attempt. The course staff will watch your progress during this recovery plan. If you do not complete the learning objective by the end of this plan, you will fail this course.

If you are placed on a recovery plan, you can expect the following

  • If you have not completed the lectures objectives by the deadline, and the course staff can see enough effort to determine that you've been spending >=12 hours/week on the course, you will get a deadline of 1 week after the checkpoint. To help us make this determination, please submit to AutoLab early and often! If your first submission is on the day of the checkpoint, we cannot determine that you've put effort into the project/course. However, if you make regular submissions with steady progress start at, or before, the project prep lab then you'll almost certainly qualify for an extension.
  • You have the second chance lab the week after each checkpoint as an opportunity to makeup the read and explain portions of the objective.
  • If you do not complete the read/quiz and explain/interview portions during the second chance lab, you will still have 2 more opportunities to complete them, both of which will be scheduled outside of lab. The first (third overall) opportunity will be with a another TA and your last opportunity will always be with me (Jesse). I will make all final decisions when a student is failing after given all these opportunities.
  • Makeup quizzes, after the first 2 in lab, will be oral. You will be shown a program and asked to explain how it works while answering followup questions.

Application Objectives

You can improve your letter grade by showing that you can apply the learning objectives to solve real-world problems. You will have the following opportunities to complete application objectives.

  • Programming Projects
    • Each of the 4 projects will contain 2 objectives that will be labeled as Application Objectives. Each of these is worth an application objective in the course for a total of 8 available AOs.
    • The intent is for the first AO on a project to be reasonably obtainable, while the second offers a more significant challenge.
    • All programming AOs are due at the learning objective checkpoint. There is a "get it on my desk by the morning" deadline for programming AOs of 9am the morning after the checkpoint. There are no deadline extensions for AOs.
  • Final Exam
    • The final exam will offer an opportunity to earn up to 2 applications objectives and improve your grade. For the final exam, you will be provided several English descriptions of project ideas. Your task is to explain how you would design and write software for the project using the concepts you've learned in this course.

In addition to the department and university policies, the following details apply to this course.

All submitted work must be of your own creation and you must not share your submission with anyone else. If any submission is very similar to code that has been submitted by another student, or can be found online, it is in violation of this courses academic integrity policy and all students will be penalized whether they were copying or sharing their code with other students so they can copy. If two submissions are similar beyond what is likely if the students worked independently, then both students are in violation of the academic integrity policy.

All violations will result in:

An F in CSE116.

It is your responsibility to understand what constitutes an academic integrity violation. If you have any question whether something you are doing is a violation or not, ask for clarification before receiving an F in the course. I will not entertain excuses after you have been caught.

Examples of acceptable behavior:

  • Discussing an assignment with your classmates and brainstorming abstract solutions, then writing code independently
  • Forming study groups and discussing course material and assignments on Discord, Zoom, etc.
  • Searching the Internet for supplementary material on the course topics
  • Asking the teaching staff for clarification on a homework question
  • Having, or hiring, someone to help you understand the concepts of the course

Examples of unacceptable behavior:

  • Pressuring a TA to falsify an interview report or application objective verification.
  • Submitting code or other work, for course credit, that is not your own.
  • Collaborating with another student to write code for an assignment. <-- This is the most common type of violation. DO NOT write code together!
  • Allowing another student to see your code or submission for an assignment.
  • Allowing another student to access your work that will be submitted for course credit. (Examples: Do not post you code publicly, host your code in a public repository, or allow a classmate to access your laptop)
  • Having, or hiring, someone to help you write code
  • Copying a large amount of material found on the Internet into your submission.
  • Attempting to access the grading code on AutoLab.

If you plan on cheating, plan on taking this course again.
