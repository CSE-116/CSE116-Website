First Application Objective - Debugging

Complete the following debugging requirements to earn your first application objective on this assignment.

Testing Requirements

Write tests in the tests.TestDebug1 class for the following functionality:

  • A the method named standardDeviation in the debug1.StandardDeviation class. The starter code provides an implementation for this method that is incorrect. Your tests should expose any and all bugs in this implementation. The correct behavior of this method should have the following features:
    • The standardDeviation method takes an ArrayList of Doubles and returns a double that is the standard deviation of the values in the input
    • The method should calculate the population standard deviation (Not sample standard deviation)
    • If the input is an empty ArrayList, the method should return 0.0

Write tests in the tests.TestDebug2 class for the following functionality:

  • A the method named findKeys in the debug2.KeyFinder class. The starter code provides an implementation for this method that is incorrect. Your tests should expose any and all bugs in this implementation. The correct behavior of this method should have the following features:
    • The findKeys method takes two HashMaps of String to Integer then an int, and returns an ArrayList of Strings. The returned ArrayList will contain all the keys that map to the int from the parameter list in either of the HashMaps. This ArrayList will be sorted by the length of the String, with ties broken alphabetically ignoring case
    • If a key maps to the given int in both maps, it should only appear in the output once
    • If a key maps to the given value in one of the HashMaps and is in the other HashMap mapping to a different value, it should still be in the returned ArrayList

Programming Requirements

Use your tests and the debugger to correct the implementations of the 2 tested methods.

Second Application Objective - Testing with Mocks

Complete the following requirements to earn your second application objective on this assignment.

Testing Requirements

In the mocks package, you are provided a Game class which keeps track of scores for players and can find the player with the highest score. Unlike the other code provided in the repo, this code functions properly but your objective is still to test this code. The game class uses a Database for its functionality that connects to a MongoDB database server. You may install MongoDB if you'd like to run this code, but that is not necessary for this assignment and there will not be a MongoDB server running in AutoLab during grading. Your task is to test the Game class functionality using a mock implementation for the Database.

  • You need to test the following functionality of the mocks.Game class:
    • A constructor that takes a reference to a Database object that will be accessed by this class for functionality
    • A method named scorePoints that takes the name of a player as a String, a number of points as an int, and returns void. The method will use the Database retrieve the current score of the player, then add the newly scored points to that score and store the new score in the Database
    • A method named getWinner that takes no parameters and returns the name of the player with the highest score. If multiple players are tied for the highest score, any one of them will e returned and your test should accept any of the tied players as correct

The mocks.Game is complete and does not need to be changed for this assignment. Your task is only to test this class. However, you won't have access to a MongoDB server while testing and so you must set up a mock database to test this class.

  • To set up a mock database, implement the following:
    • Create a mock database class in your tests package that implements the Database interface. It's important that this class is in your tests package since only code in your tests package is used when running your tests against correct/incorrect solutions in AutoLab
    • Pass an instance of your mock database to the constructor of the Game class which allows you to test the Game class without running a database server. You may choose any method of implementing the three Database methods that you'd like
Programming Requirements

There are no programming requirements outside of code you've written in your tests package. Everything in the mocks package can remain unchanged from the provided code
